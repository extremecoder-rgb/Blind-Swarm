<task type="auto">
  <name>Create BlindSwarm.compact contract</name>
  <files>contracts/BlindSwarm.compact, contracts/index.ts</files>
  <action>
Create BlindSwarm.compact with the following state and circuits:

**State Definition:**
- AgentMap: Map<AgentId, Agent> - stores agent ID, owner, capabilities, stake, reputation
- TaskMap: Map<TaskId, Task> - stores task DAG, statuses, escrow, step details  
- DisputeMap: Map<DisputeId, Dispute> - stores disputes, evidence, resolution state

**Circuits (ZK State Transitions):**
1. register_agent(owner: Address, capabilities: [Capability], stake: u64)
   - Validates stake > minimum
   - Generates unique agent ID
   - Emits AgentRegistered event

2. deregister_agent(agent_id: AgentId)
   - Validates agent exists and caller is owner
   - Releases stake
   - Emits AgentDeregistered event

3. create_task(dag: DAG, escrow_amount: u64, deadline: u64)
   - Validates DAG is acyclic (no cycles)
   - Validates all steps have valid dependencies
   - Locks escrow amount
   - Emits TaskCreated event

4. assign_step(task_id: TaskId, step_index: u32, agent_id: AgentId)
   - Validates task exists and step is pending
   - Validates agent is registered
   - Updates step assignment
   - Emits StepAssigned event

5. submit_attestation(task_id: TaskId, step_index: u32, signature: Signature, output_hash: Hash)
   - Validates signature from assigned agent
   - Validates output_hash matches computation
   - Updates step status to completed
   - Emits AttestationSubmitted event

6. initiate_dispute(task_id: TaskId, step_index: u32, evidence_commitment: Hash)
   - Validates step was completed
   - Creates dispute with evidence commitment
   - Emits DisputeInitiated event

7. resolve_dispute(dispute_id: DisputeId, resolution: Resolution, selective_data: [Field])
   - Validates selective disclosure proves correctness
   - Updates dispute status
   - Releases or slashes escrow based on resolution
   - Emits DisputeResolved event
  </action>
